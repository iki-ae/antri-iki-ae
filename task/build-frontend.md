## AUTH ‚Äî AXIOS INTERCEPTOR

`api/index.ts` intercepts 401 and redirects to login for mid-session token expiry. **Critical rules:**

- **Never redirect on `/auth/me` or `/auth/logout` 401s** ‚Äî these are expected to 401 when unauthenticated (bootstrap/teardown). Redirecting on them causes an infinite loop.
- **Never redirect when already on a public route** (`/display`, `/kiosk`, `/login`) ‚Äî these pages have no auth requirement and should load regardless.
- **Use `store.clear()` not `store.logout()`** ‚Äî `logout()` calls `/api/auth/logout` which itself 401s unauthenticated, re-entering the interceptor. `clear()` resets state only, no API call.

```typescript
const PUBLIC_ROUTES = ['/display', '/kiosk', '/login']
const SKIP_REDIRECT_URLS = ['/auth/me', '/auth/logout']

api.interceptors.response.use(null, (err) => {
  const url: string = err.config?.url ?? ''
  if (err.response?.status === 401 && !SKIP_REDIRECT_URLS.some(u => url.includes(u))) {
    import('@/router').then(({ default: router }) => {
      const current = router.currentRoute.value.path
      if (PUBLIC_ROUTES.some(p => current.startsWith(p))) return
      import('@/stores/auth').then(({ useAuthStore }) => useAuthStore().clear())
      router.replace('/login')
    })
  }
  return Promise.reject(err)
})
```

The auth store exposes `clear()` (state reset, no API call) and `logout()` (calls `/api/auth/logout` then `clear()`). Only the interceptor uses `clear()` directly.

## SSE ‚Äî CLIENT SIDE

- `queue.ts` Pinia store subscribes to `GET /api/events`
- On message: parse `QueueState`, update store
- Display, Kiosk, and Operator views all use this store ‚Äî no polling anywhere
- Reconnect on network drop ‚Äî re-subscribe, get full state immediately

**queue.ts extended API (display page use only):**
- `onAnnounce(fn)` / `offAnnounce(fn)` ‚Äî fires synchronously per SSE message (before Vue batches) for each counter whose `calledAt` changed. Use this for announcement queuing ‚Äî never `watch(lastCalled)` which only sees the final batched value.
- `onFirstState(fn)` / `offFirstState(fn)` ‚Äî fires once on the first (seed) SSE message after connect. Use to initialise display state silently without triggering sound. Resets on disconnect.
- The first SSE message after connect is always a silent seed ‚Äî it populates `_lastCalledAt` and fires `onFirstState` but does NOT fire `onAnnounce`. This prevents page-load state from being treated as new calls.

## INPUT FIELD PATTERN

Do **not** use `ion-input` for form fields. Use native `<input>` with CSS floating labels.

```html
<div class="field">
  <input id="x" type="text" placeholder=" " />
  <label for="x">Label Text</label>
  <span class="icon"><!-- SVG icon, right-aligned --></span>
</div>
```

Rules:
- `placeholder=" "` (single space) is required ‚Äî the CSS float trigger depends on `:not(:placeholder-shown)`
- Label floats up on focus or when filled via: `input:focus ~ label, input:not(:placeholder-shown) ~ label`
- Floated state: `font-size: 10.5px`, `font-weight: 600`, `color: var(--color-accent)`, `text-transform: uppercase`
- Input height: `56px`, padding: `18px 44px 6px 16px` (top clears the floated label, right clears the icon)
- Focus ring: `border-color: var(--color-accent)` + `box-shadow: 0 0 0 3px rgba(224,140,47,0.12)`
- Icon slot: `position: absolute; right: 14px; top: 50%; transform: translateY(-50%)`
- Password toggle icon uses `pointer-events: all` and swaps SVG via `v-if/v-else` on `showPassword` ‚Äî no JS DOM manipulation

## TICKET PRINTING

**`src/utils/print.ts`** ‚Äî the single print entry point. Two exports:
- `printSlips(tickets, docTitle, institutionName)` ‚Äî replaces the current document with a self-contained slip HTML page, triggers print, then navigates back to `/kiosk` after a delay.
- `printSingleKiosk(data, institutionName)` ‚Äî wraps a single kiosk ticket as `PrintTicket` and calls `printSlips`.

**Architecture ‚Äî `document.open()` replacement:**
`document.open('text/html', 'replace')` + `document.write(html)` + `document.close()` completely replaces the live document, destroying all Ionic web components and shadow DOM. This is the only approach that works reliably on Android Chrome with a Bluetooth thermal printer ‚Äî popup windows and iframes both fail (popup hangs on "Preparing preview"; iframe captures the main window).

The replaced document contains all HTML/CSS/JS inline. `window.print()` is called from `window.onload` inside the injected `<script>` ‚Äî runs in the new document's context after full layout.

**Navigation back to kiosk:**
- `window.onafterprint` ‚Üí `setTimeout(goKiosk, 5000)` ‚Äî 5s delay lets the ESC/POS service finish receiving before the page changes (navigating too fast causes the printer to capture the new page)
- Fallback: `setTimeout(goKiosk, 30000)` set on `window.onload` ‚Äî fires if `onafterprint` never fires
- `clearTimeout(fallback)` inside `onafterprint` prevents double-navigation
- Manual "‚Üê Klik di sini" button always visible (hidden behind OS dialog during normal flow) as last resort

**Visitor screen while printing:**
Dark background (`#0a0a1a`), ticket number in orange, "Tiket sedang dicetak...", "Halaman akan kembali otomatis dalam 30 detik". Slip content is `display: none` on screen ‚Äî only `display: block` in `@media print`.

**`PRINT_CSS` (inline in `print.ts`):**
- `@page { size: 55mm 80mm; margin: 0 }` ‚Äî paper width 55mm, height 80mm
- `.slip-page + .slip-page { page-break-before: always }` ‚Äî page break between slips only, no trailing blank page
- Slip content classes: `.slip-institution`, `.slip-session`, `.slip-category`, `.slip-divider` (dashed hr), `.slip-number` (24pt bold), `.slip-issued`, `.slip-qr`, `.slip-watermark`

**Slip layout** (top to bottom, all centered):
1. Institution name ‚Äî 8pt bold
2. Session title ‚Äî 6.5pt
3. Category prefix ‚Äî category name ‚Äî 6.5pt
4. Dashed `<hr>`
5. Display number (e.g. `A-007`) ‚Äî 24pt bold
6. Dashed `<hr>`
7. Issued date+time (`created_at` formatted `DD/MM/YYYY HH:mm` in `id-ID` locale)
8. QR code ‚Äî 16√ó16mm inline SVG (generated by `qrcode-generator`); payload: `{institutionName} | {display_number} | powered by iki.ae`
9. `powered by iki.ae` ‚Äî 6pt gray

**QR generation:** `import qrcode from 'qrcode-generator'` ‚Äî called at print time in `slipHtml()`, returns inline SVG string. No network request, works offline.

**Trigger surfaces:**
- **Kiosk** (`KioskPage.vue`): auto-print on every `take()` ‚Äî `printSingleKiosk()` called after API response. No countdown timer ‚Äî ticket result stays on screen until next visitor taps a category.
- **SessionPage** (`SessionPage.vue`): print button (blue printer icon) on every session row, disabled when `issued === 0`. Print modal:
  - Kiosk session: single number input (zero-padded, e.g. `007`), validates `1 ‚â§ n ‚â§ issued`.
  - Bulk session: from/to range inputs (zero-padded), validates `1 ‚â§ from ‚â§ to ‚â§ issued`.
  - On confirm: `ticketsApi.bySession(sessionId, from, to)` ‚Üí `executePrint()` (aliased import to avoid name collision).

**`app.name` i18n + `<title>`**: both set to `antri-iki-ae` (all lowercase, hyphenated).

**Known constraints:**
- Chrome ignores `@page { size }` when saving to PDF ‚Äî physical thermal printer drivers handle roll width correctly.
- `onafterprint` fires before ESC/POS service finishes on Android ‚Äî always delay navigation by ‚â•5s after it fires.
- `document.open()` destroys Vue reactivity ‚Äî the app reloads via `location.replace('/kiosk')` to restore state. This is correct for kiosk; admin SessionPage print also reloads (acceptable ‚Äî admin can re-navigate).

**Key rule:** Never use `window.open()` popup or hidden `<iframe>` for mobile/Android printing ‚Äî both fail. Always use `document.open()` replacement on the current tab.
